Why do we use a `.venv` virtual environment in Python projects, and what issues does it prevent?

## Reference Answer

A `.venv` gives each project its own Python environment.
That keeps project dependencies isolated from global Python.
Different projects can use different versions safely without conflicts.
It improves reproducibility because others can recreate the same setup.
It also keeps systems cleaner and makes troubleshooting easier.
